# 포팅 이슈 형식 (단일 소스)

h5-port STEP 3-B와 포터(pureweb-porter·toss-porter)가 포팅 이슈를 생성할 때 이 파일을 Read해 형식을 따른다.
형식 정의는 이 파일이 유일하다 — 다른 문서에 본문을 복사해 두지 않는다(드리프트 방지).

## 제목 형식

```
[포팅] {PLATFORM} — {프로젝트명}
```

- `{PLATFORM}`: `pureweb` 또는 `toss`
- `{프로젝트명}`: 프로젝트 루트 폴더명

기존 이슈 검색도 같은 접두사를 쓴다: `gh issue list --state open --search "[포팅]"` — open 이슈가 이미 있으면 새로 만들지 않고 그 번호를 재사용한다.

## 본문 형식

```markdown
## 진행 상황
정본: 기반(컴파일/런타임/공백) 이슈는 항상 `Docs/porting/pureweb-checklist.md`, 플랫폼 전용 이슈는 `Docs/porting/{PLATFORM}-checklist.md` (이 이슈는 진행 미러 — 상세 상태는 두 정본 파일 참조)

{체크리스트 `## 단계 진행` 스냅샷 — 아직 없으면 "(포터가 체크리스트 생성 후 반영)"}

## 사람 준비 항목 (토스 포팅만 — 퓨어웹이면 이 섹션 삭제)
확정된 항목은 `[x]` + 확정값 기재, 미확정이면 `[ ]`로 남긴다. 해당 없는 항목은 줄을 지운다.
- [ ] 배너 위치 (상단/하단):
- [ ] IAP PID 매핑:
- [ ] 햅틱 타입:
- [ ] 공유하기 문구:
- [ ] 프로모션 ID:
- [ ] 랭킹 버튼 위치:

## 확인 필요 / 미확정
(포터 실행 중 항목이 추가되면 여기 기록됨)
```

## 작성 규칙

- **h5-port STEP 3-B에서 생성하는 경우** (토스): `## 사람 준비 항목`에 STEP 3-A에서 수집한 답변을 반영한다 — 확정 `[x]` + 값, 미확정 `[ ]`, 해당 없음은 줄 삭제.
- **포터가 직접 생성하는 경우** (prompt에 이슈 번호 없이 단독 실행): STEP 3-A 답변이 없으므로 `## 사람 준비 항목`은 전부 `[ ]` 미확정으로 생성한다 — 포터의 기존 미확정 폴백(제안 삽입+검토 기록)이 그대로 적용되고, h5-port 후속 모드(재실행 시 미체크 항목 재질문)가 나중에 수집한다.
- 본문은 heredoc 임시 파일로 전달해 서식이 깨지지 않게 한다 (`gh issue create --body-file`).
