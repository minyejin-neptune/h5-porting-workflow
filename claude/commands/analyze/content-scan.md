# 콘텐츠 연관 그래프 스캔

$ARGUMENTS로 지정한 콘텐츠의 코드 연관을 추적해 그래프를 생성한다.

이 스킬은 `content-analyze`가 콘텐츠를 분석하기 전에 실행하는 선행 스캔이다. `porting-scan → PORTING_VOCAB.md` 관계와 동일하게, `content-scan → {콘텐츠명}_연관그래프.md` 관계로 동작한다 — 한 번 만든 그래프는 재사용되고, `content-analyze`는 이 그래프를 읽어서 재탐색 없이 문서를 쓴다.

추론 금지. 코드에서 확인한 사실만 기재. 확인 불가 시 "확인 필요" 표시.
출처는 파일명:라인번호 형식으로 명시.
**하드코딩 금지** — 이 스킬 문서 자체에 특정 프로젝트의 함수·클래스명을 넣지 않는다(예시 제외). 관계 유형(카테고리)은 `docs/content-relation-mapping-rules.md`를 참조한다 — 여기서 재정의하지 않는다.

모델: 세션 기본값(`sonnet`) 사용, effort 조정 없음 — grep/Read 확인 위주의 기계적 탐색이라 기본값으로 충분 (analyze:content-analyze의 분석·문서화 단계는 `effort: max`를 쓰는 것과 대비됨).

인자가 비어 있으면 멈추고 "분석할 콘텐츠명을 인자로 입력해주세요 (예: `/analyze:content-scan 스킬`, 영문 식별자를 알면 `/analyze:content-scan 스킬(Skill)`)" 라고 안내한다.

`--base-commit {해시}` 플래그를 함께 받으면(예: `/analyze:content-scan 스킬 --base-commit abc1234`) STEP 1의 `기준 커밋` 자동 감지를 생략하고 이 값을 그대로 쓴다 — 콘텐츠명 파싱에서는 이 플래그와 그 값을 제외한 나머지를 콘텐츠명으로 취급한다.

---

## STEP 0 — 신선도 판정

```bash
GRAPH_FILE="Docs/design/{콘텐츠명}/{콘텐츠명}_연관그래프.md"
ls "$GRAPH_FILE" 2>/dev/null && echo "EXISTS" || echo "NONE"
```

- **없음** → STEP 1부터 생성
- **있음** → 신선도를 직접 판정한다 (sdk-list-analyzer와 동일한 방식):
  ```bash
  BASE_COMMIT=$(grep -oE "기준 커밋: [0-9a-f]+" "$GRAPH_FILE" | awk '{print $3}')
  { git diff --name-only "$BASE_COMMIT" HEAD -- '*.cs' 2>/dev/null;
    git status --porcelain -- '*.cs' 2>/dev/null | awk '{print $2}'; } | sort -u
  ```
  - 출력 0건 (신선) → "기존 그래프 신선 — 재사용 (기준 커밋 {해시 앞 7자리})"을 출력하고 종료한다. **재탐색하지 않는다.**
  - 출력 1건 이상 (낡음) → 변경 파일 목록을 출력하고 STEP 1부터 재생성한다 (기존 파일 덮어씀)
  - `BASE_COMMIT` 추출 실패 또는 git repo 아님 → 판정 불가 — 재생성한다

## STEP 1 — 프로젝트 설정 자동 감지

| 설정 항목 | 감지 방법 |
|---|---|
| `SCRIPTS_PATH` | `Docs/porting/PORTING_VOCAB.md`가 존재하면 해당 파일의 `{SCRIPTS_PATH}` 행 값을 사용. 없으면 `find . -maxdepth 4 -type d -iname "*script*"` 결과 중 가장 포괄적인 경로 |
| `UI_PATH` | SCRIPTS_PATH 하위 `UI` 또는 `ui` 디렉토리 |
| `DATA_FILE` | 1차: `find . -path "*/Resources/*" \( -iname "*data*" -o -iname "*table*" \) -type f 2>/dev/null` — 0건이면 STEP 4 Zero-Hit Fallback 절차를 적용, 그래도 못 찾으면 "확인 필요" (특정 파일명을 하드코딩하지 않는다) |
| `DATA_DECODE_CMD` | `Tools/` 디렉토리 또는 CLAUDE.md에서 디코딩 도구 확인 — 없으면 "직접 파싱" |
| `기준 커밋` | `--base-commit` 인자가 있으면 그 값 사용. 없으면 `Docs/porting/NATIVE_BASELINE.md`가 존재할 때 그 헤더의 `기준 커밋` 값을 재사용(같은 프로젝트 내 산출물들이 서로 다른 기준점을 갖지 않도록). 둘 다 없으면 `git rev-parse HEAD 2>/dev/null \|\| echo "NO_GIT"`로 직접 캡처 |

감지 결과를 사용자에게 한 번에 보여주고 확인받은 뒤 진행한다. 이 값들(SCRIPTS_PATH·DATA_FILE·DATA_DECODE_CMD·기준 커밋)은 STEP 5 산출물 헤더에 그대로 기록한다 — `content-analyze`가 이 콘텐츠를 재분석할 때 재감지 없이 그래프 헤더에서 재사용한다.

## STEP 2 — 콘텐츠명 → 식별자 후보 생성 + 1차 노드 추출

### 2-1. 식별자 후보 생성

콘텐츠명(한글)을 코드 식별자 후보로 변환한다:
- 사용자가 영문을 괄호로 병기했으면(`스킬(Skill)`) 그 값을 그대로 채택 — 후보 생성 생략
- 없으면 일반적 로마자 표기 + 흔한 대응 영단어를 후보로 생성한다 (예: "스킬"→`Skill`, "재화"→`Currency`/`Gold`/`Coin`, "펫"→`Pet`). 후보는 최대 3~4개로 제한한다.
- 후보 각각에 대해 아래 1차 탐색을 시도하고, 히트가 나오는 첫 후보를 채택한다. 전부 0건이면 STEP 4로 진행한다.

### 2-2. 1차 탐색

```bash
find {SCRIPTS_PATH} -iname "*{후보}*.cs" 2>/dev/null
grep -rln "class.*{후보}\|enum.*{후보}" {SCRIPTS_PATH} --include="*.cs" 2>/dev/null
```

DATA_FILE이 있으면(DATA_DECODE_CMD로 디코딩 가능한 경우) 그 안에서도 콘텐츠명 관련 항목을 탐색한다.

히트한 파일·클래스·enum이 **1차 노드**다. 여기서 이전 버전의 하드코딩된 시스템별 grep(`IsOpen`/`redDot`/`Board` 등 특정 프로젝트 식별자)은 절대 쓰지 않는다 — 카테고리 판별은 STEP 3에서 매핑 규칙 문서를 그대로 적용한다.

---

## STEP 3 — 노드 확장 (참조 추적)

1차 노드를 Read해서 실제 참조 대상을 따라가며 그래프를 넓힌다. **엣지 종류는 `docs/content-relation-mapping-rules.md`의 카테고리(A~J, G-1/G-2)를 그대로 쓴다 — 여기서 새 카테고리를 만들지 않는다.**

각 1차 노드 파일에 대해 매핑 규칙 문서의 "코드 근거 유형" 열에 적힌 탐지 방법을 순서대로 적용한다:

| 카테고리 | 탐지 방법 요약 (전체는 매핑 규칙 문서 참고) |
|---|---|
| A 테이블 타입 연결 | 관련 enum + 타입 필터 코드 |
| B 조건 함수/트리거 | 조건부 함수 호출·플래그 참조·**진행도 스탯 수치 비교**(`>`,`>=`,`<`) |
| C 테이블 존재 연동 | 다른 시스템 테이블에 이 콘텐츠 관련 항목 존재 |
| D 로그 이벤트 | 로그 전송 함수 호출부에 콘텐츠 식별자 인자 |
| E 메인 루프 진입 | 게임 진입점/메인 매니저의 직접 호출 또는 이벤트·delegate 연결 |
| F 실결제 IAP | 스토어/상품 테이블 컬럼 참조 또는 전용 IAP 팝업 |
| G-1 자동 노출 팝업 | 팝업 호출이 조건부 분기 안에서 발생 |
| G-2 메뉴 진입형 팝업 | 팝업 호출이 버튼 클릭 핸들러에서만 발생 |
| H 버프/모디파이어 | 다른 시스템의 계산식이 이 콘텐츠 데이터 필드를 직접 읽음 |
| I 저장 데이터 포함 | 저장/로드 데이터 구조체에 이 콘텐츠 필드가 포함 |
| J 치트/디버그 연동 | 치트 매니저가 이 콘텐츠 데이터 필드를 직접 읽거나 씀 |
| K 알림 뱃지 연동 | `{콘텐츠명}Notice`류 GameObject를 조건에 따라 SetActive로 토글 |

발견되는 각 연관을 카테고리 ID + 대상(다른 콘텐츠 노드 **또는 진행도 스탯 같은 코어 상태값**) + 파일:라인으로 기록한다.

공통 참조 제외 규칙(매핑 규칙 문서 "공통 참조 제외 규칙" 절: 프레임워크 베이스 클래스, 범용 UI 피드백 등)에 해당하면 엣지로 기록하지 않는다.

**확장 깊이 제한**: 1차 노드에서 직접 연결된 노드까지 확장한다(깊이 2). 이미 방문한 파일은 재확장하지 않는다(순환 방지). 깊이 2 노드가 강한 연관(카테고리 H·I처럼 데이터를 직접 공유)을 보이면 그 노드까지만 1단계 더 확장하고 멈춘다. 그 이상은 확장하지 않는다.

**미분류 강한 연관 처리 (버리지 않는다)**: 확장 중 A~K 어디에도 맞지 않지만 명확한 참조 관계(직접 함수 호출, 데이터 필드 공유, 조건부 트리거 등 — 우연한 참조가 아니라 구조적 연결)를 발견하면 스킵하지 않는다. A~K 목록은 지금까지 확인된 패턴의 기록일 뿐 완전한 분류 체계가 아니다 — 처음 보는 클래스·시스템이라고 그냥 지나치지 않는다. `[신규후보]` 태그로 그래프에 그대로 기록한다: 대상 시스템명 + 근거 파일:라인 + 기존 카테고리(A~K)에 안 맞는 이유 한 줄. 여기서 새 카테고리를 정식으로 만들지는 않는다 — 그건 Phase 1 매핑 규칙(`docs/content-relation-mapping-rules.md`) 갱신 몫이다. 다른 콘텐츠 스캔에서도 같은 `[신규후보]` 패턴이 반복되면 정식 카테고리 승격을 검토한다(K 카테고리가 이렇게 추가됐다).

## STEP 4 — Zero-Hit Fallback

STEP 2의 1차 노드 추출이 전체 0건이거나, STEP 3의 특정 카테고리 탐색이 0건이면:

1. **매핑 규칙 문서의 "확인 필요 처리 경계" 3단계를 그대로 적용**한다:
   - 관계 대상 시스템 자체가 프로젝트에 존재하는지 먼저 `find`로 확인 → 존재하지 않으면 "없음"으로 기록 (근거: 시스템 부재 확인됨)
   - 시스템은 있는데 이 콘텐츠와의 연계만 0건이면, 그 시스템의 정의부(함수 정의 위치)까지 Read해서 간접 경로가 없는지 1단계 더 확인
   - 그래도 판정 불가하면 `확인 필요`로 남긴다. **"없음" 임의 단정 금지.**
2. 1차 노드 추출 자체가 0건인 경우, 도메인 어휘로 재탐색한다(최대 5개 파일, 1단계 — porting-scan과 동일한 제한):
   ```bash
   grep -rilnE "{콘텐츠명 관련 도메인 어휘}" {SCRIPTS_PATH} --include="*.cs" 2>/dev/null \
     | grep -vE "Test|Editor" | head -5
   ```
   히트 파일을 Read해 확인되면 노드에 `(fallback 확인: 파일:라인)` 표기. 확인 안 되면 `확인 필요 — fallback 후보: {파일목록}`으로 기록.

## STEP 5 — 그래프 산출물 저장

`Docs/design/{콘텐츠명}/` 디렉토리가 없으면 `mkdir -p` 후 저장한다. 형식은 마크다운 **중첩 목록**(표 아님):

```markdown
# {콘텐츠명} 연관 그래프

> 생성: {날짜} (content-scan) | 기준 커밋: {커밋 해시 | NO_GIT} | SCRIPTS_PATH: {경로} | DATA_FILE: {경로 | 없음} | DATA_DECODE_CMD: {명령 | 직접 파싱}

- **{콘텐츠명}** (`{대표 파일:라인}`)
  - [A] {관련 테이블/enum} — `{파일:라인}`
  - [B] {조건 대상: 다른 콘텐츠명 | 진행도 스탯명} — `{파일:라인}`
  - [H] {영향받는 시스템} — `{파일:라인}`
    - [A] {그 시스템의 하위 연관 노드} — `{파일:라인}` (깊이 2 확장)
  - [확인 필요] {카테고리} — {판정 불가 사유}
  - [신규후보] {대상 시스템명} — `{파일:라인}` (기존 카테고리 미해당 사유: ...)
```

각 노드는 카테고리 ID(대괄호)로 태깅하고, 근거 파일:라인을 반드시 포함한다.

## STEP 6 — 완료 출력

최종 텍스트로 아래만 출력한다:

```
✅ Docs/design/{콘텐츠명}/{콘텐츠명}_연관그래프.md 생성 완료 — 노드 N개, 카테고리 M종, 확인 필요 K건, 신규후보 L건
기준 커밋: {해시 앞 7자리 | NO_GIT}
```
