# 작업 세분화 (Feature Breakdown)

아래 문제를 **독립적으로 실행 가능한 하위 작업 목록**으로 세분화한다.

> 대상 문제: **$ARGUMENTS**

인자가 비어 있으면 여기서 멈추고 "세분화할 문제를 인자로 입력해주세요 (예: `/common:feature-breakdown 로그인 API에 소셜 로그인 추가`)" 라고 안내한다.

---

## 원칙 (엄수)

- **추론 금지 표기**: 코드·문서에서 근거를 확인한 하위 작업은 파일:라인 근거를 붙인다. 근거 없이 필요하다고 판단한 작업은 반드시 `[추정]` 태그를 달고, 확인이 필요하면 `[확인 필요]`로 표기한다.
- **범위 고정**: 입력한 문제를 푸는 데 필요한 작업만 낸다. "하는 김에" 식의 리팩터링·기능 확장은 별도 `범위 밖(선택)` 절로 분리한다.
- **불충분하면 질문**: 문제 정의가 모호해 기준을 못 잡으면, 임의로 가정하지 말고 먼저 1~3개 확인 질문을 던진다.

---

## Step 1 — 분해 기준(축) 결정

문제 성격에 맞는 축을 **하나 골라 명시**하고 그 이유를 한 줄로 적는다. 여러 축이 필요하면 주 축 + 보조 축으로 정리한다.

- **기능 단위** — 서로 다른 기능/모듈로 나뉠 때
- **파이프라인 단계** — 입력→처리→출력처럼 순서가 있는 변환일 때
- **파일·모듈 단위** — 손댈 코드 위치가 분명할 때
- **의존성 순서** — 선행 작업이 뒤 작업을 막을 때

## Step 2 — 하위 작업 도출

각 하위 작업을 아래 4요소를 채워 정의한다. 하나라도 못 채우면 작업 경계가 잘못된 것이니 다시 쪼개거나 합친다.

| 요소 | 설명 |
|---|---|
| **ID** | `T1`, `T2` … (추적·병합용) |
| **작업** | 무엇을 하는가 (동사로 시작) |
| **입력** | 시작에 필요한 것 (파일·데이터·선행 결과) |
| **출력** | 끝났을 때 나오는 산출물 |
| **완료 조건(DoD)** | 무엇을 보면 "됐다"고 판단하는가 (검증 가능하게) |

### 粒度(입도) 규칙 — 중지 조건
- 한 하위 작업 = **단일 파일 수정 또는 단일 검증 항목** 수준까지 쪼갠다.
- 더 쪼개도 실행 방법이 같으면 **멈춘다**. (과분해 금지)
- 한 작업이 여러 파일·여러 판단을 요구하면 **더 쪼갠다**.
- 재귀 깊이는 기본 **2단계**까지. 그 이상 필요하면 사용자에게 알린다.

## Step 3 — 의존성·실행 순서

- 작업 간 선후 관계를 표기한다: `T2 ← T1` (T1 완료 후 T2).
- **병렬 가능**한 작업과 **순차 필수**인 작업을 구분해 묶는다.
- 순환 의존(A↔B)이 생기면 경계를 잘못 그은 것 — 다시 나눈다.

## Step 4 — 완전성·중복 검증 (자기 점검)

계획을 내기 전에 스스로 검사한다:
- **완전성**: 하위 작업의 합집합이 원래 문제 전체를 덮는가? 빠진 조각은 없는가?
- **중복**: 두 작업이 같은 산출물을 만들지 않는가? 겹치면 합친다.
- **독립성**: 병렬로 표시한 작업들이 정말 서로의 결과 없이 시작 가능한가?

빠진 부분이 있으면 작업을 추가하고, 못 덮는 영역이 있으면 `[미해결/확인 필요]`로 남긴다.

## 출력 포맷 (고정)

아래 형식 그대로 출력한다. 다음 단계(사람 검토 또는 하위 에이전트 위임)가 그대로 소비할 수 있어야 한다.

```
## 문제
<$ARGUMENTS 재진술 — 한 줄>

## 분해 기준
- 주 축: <축> — <이유>
- (보조 축: <축>)

## 하위 작업
### T1 — <작업명>
- 입력: <...>
- 출력: <...>
- 완료 조건: <...>
- 의존: <없음 | T… 이후>
- 근거: <파일:라인 | [추정] | [확인 필요]>

### T2 — <작업명>
- ...

## 실행 순서
- 병렬 그룹 A: T1, T3
- 순차: T2 ← T1, T4 ← T2

## 완전성 점검
- 덮이지 않은 영역: <없음 | ...>
- 중복 제거: <없음 | T… 와 T… 통합>

## 범위 밖(선택)
- <하는 김에 할 만하지만 이번 문제에 불필요한 것 — 있으면>
```

---

## 마무리

계획만 산출하고 **여기서 멈춘다**. 실제 작업 수행이나 하위 에이전트 위임은 사용자가 계획을 확인한 뒤 별도로 지시할 때 진행한다.
