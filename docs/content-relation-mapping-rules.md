## 공통 참조 제외 규칙 (노이즈 방지)

아래에 해당하면 엣지로 취급하지 않는다:

1. **프레임워크 베이스 클래스/네임스페이스** — `MonoBehaviour`, `System.Collections`, `UnityEngine.UI` 등 언어·엔진 기본 요소
2. **범용 UI 피드백** — 콘텐츠 특정적이지 않고 프로젝트 전반의 임의 버튼·패널에서 동일하게 반복되는 공통 연출 호출(예: 공통 사운드 재생 함수)
3. 판정 기준: 참조 대상이 **(a)** 이 콘텐츠 없이도 다른 임의의 시스템에서 동일한 방식으로 등장하고, **(b)** 이 콘텐츠의 실제 상태(데이터값)를 읽거나 쓰지 않으면 → 제외

> 파일럿 근거: `GameSounds.GameSound.Instance.Play(...)` (`SkillManager.cs:103,126`) — 버튼 클릭 시 범용 사운드 재생, 콘텐츠 상태를 읽지도 쓰지도 않음 → 제외 대상 사례

## `확인 필요` 처리 경계

코드 근거가 없다고 해서 곧바로 "연관 없음"으로 단정하지 않는다. 판정 순서:

1. **관계 대상 시스템 자체가 프로젝트에 존재하는지 먼저 확인** (`find`로 파일 존재 여부) — 존재하지 않으면 "없음"으로 기록 가능 (근거: 시스템 부재 확인됨)
   - 파일럿 사례: Quest 시스템 자체가 프로젝트에 없음(`find` 0건) → C는 "없음"으로 판정
2. **시스템은 존재하는데 이 콘텐츠와의 직접 연계가 0건**이면, 그 시스템의 정의부(함수 정의 위치)까지 Read해서 간접 경로가 없는지 1단계 더 확인
   - 파일럿 사례: `SendLog`/`LogEvent` 정의 파일(`ShopManager.cs`,`GameManager.cs`,`ServiceManager.cs`)까지 확인했으나 skill 연계 없음 → D는 "없음"으로 판정 (근거: 확인한 파일 목록 명시)
3. **위 확인을 거쳤는데도 판정 불가**하거나, 코드로는 확인 불가능한 순수 기획적 연관이 의심되는 경우만 `확인 필요`로 남긴다. "없음" 임의 단정 금지.
